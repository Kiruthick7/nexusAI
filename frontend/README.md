# Nexus AI - Dashboard Frontend Module

The **Nexus AI Dashboard Frontend Module** is a premium, high-fidelity React web application built with **Next.js 15 (App Router)** and **TypeScript**. It serves as an interactive visual flight recorder and presentation console for the Nexus AI Multi-Agent Ingestion & Adjudication Platform.

---

## 🎨 Design Philosophy & Layout Boundaries (Stitch Preservation)

The interface is strictly designed, structured, and polished following the **Stitch Design System** guidelines. To ensure system presentation consistency and maintain visual design integrity, we adhere to the following rules:

- **Source of Truth**: The UI generated by Stitch is the absolute visual source of truth.
- **No Manual Redesigns**: We never modify margins, paddings, color schemes, typographies, or visual hierarchies unless addressing specific display bugs.
- **Decoupled Business Logic**: No business calculations are stored inside components. Components are strictly presentation-only wrappers that render what is published by the backend or the local simulation playbacks.

---

## 🛠️ Stack & Technologies

- **Core**: Next.js 15 (App Router, React 19)
- **Styling**: Tailwind CSS & Vanilla HSL CSS token mappings
- **State Store**: Zustand (`src/store/useStore.ts`)
- **Transitions & Micro-Animations**: Framer Motion & Lucide Icons
- **UI Components**: custom-tailored shadcn/ui primitives

---

## 🏛️ Flow Boundary & State Control

All state updates follow a unidirectional, reactive flow, decoupling components from direct data mutations:

```text
SSE Channel (FastAPI /events) ➔ EventSource Ingestion ➔ Zustand Store Actions ➔ Component Render
```

1. **State Centralization**: The global client-side state is entirely encapsulated inside the **Zustand** store (`src/store/useStore.ts`).
2. **Event-Driven Reactivity**: The client subscribes to SSE streaming channels. On receiving backend events, the store transitions its internal step progress and updates the dashboard view.
3. **No Direct Component Mutation**: Components react to state changes published by the store, preventing inconsistent views or race conditions.

## 📂 Primary Dashboard Pages

### 🎮 1. Mission Control (Primary Demo Portal)
The master dashboard visualizer that tracks claims in real-time. Features include:
- **Timeline Progression**: Captures active state milestones:
  `Intake ➔ Planner ➔ Parallel Specialists ➔ Arbiter Conflict Check ➔ Tool Gate ➔ Adjudication Decision`.
- **Specialist Lanes**: Shows concurrent validation logs for the Provider Agent, Policy Agent, and Pattern Agent.
- **Arbiter Decision Panel**: Displays aggregated evidence bundles, conflict resolution summaries, and active Tool Gate clearance states.
- **Human Escalation Widget**: Synthesizes custom reviewer questionnaires and plays spoken Text-to-Speech audio briefings directly inside the browser.
- **Gemma Audit Inspector**: Displays out-of-band explanations and logical reviews generated by the Gemma Intelligence layer.

### 📋 2. Claims History
- A clean, tabular, read-only log tracking previous adjudication runs. No inline business mutations occur here.

### 🏥 3. System Health
- Monitors backend readiness, database connectivity, and agent service states in real-time, communicating directly with backend `/health` endpoints.

### ⚙️ 4. Settings
- Enables users to configure server endpoint parameters and toggle between **Mock** (offline playback simulator) and **Live** (real-time FastAPI connection) data modes.

## 🚀 Running Locally

To start the Next.js frontend development server locally:

### 📋 Prerequisites
Ensure you have **Node.js 18+** installed.

### 💻 Local Run
1. Navigate to the frontend directory:
   ```bash
   cd frontend
   ```
2. Install node dependencies:
   ```bash
   npm install
   ```
3. Launch the Next.js development server:
   ```bash
   npm run dev
   ```
4. Open [http://localhost:3000](http://localhost:3000) inside your web browser.
