# Nexus AI Operations Platform - Frontend Rules

## Purpose
This frontend visualizes a multi-agent AI orchestration workflow. The frontend is presentation-only; business logic belongs in the backend.

---

## UI Boundaries
- The UI generated by Stitch is the source of truth.
- Never redesign pages.
- Never replace layouts.
- Never modify spacing, typography, color system, or hierarchy unless fixing bugs.
- Always preserve the Stitch design.

---

## Tech Stack
- **Next.js 15**
- **TypeScript**
- **Tailwind CSS**
- **shadcn/ui**
- **Framer Motion**
- **Lucide Icons**
- **Zustand**
- **Server Sent Events (SSE)**

---

## Architecture Flow
Components should never directly call APIs. The data flow MUST adhere to:
`API ➔ Service ➔ Store ➔ Components`

---

## State & Business Logic
- Global state belongs in Zustand.
- Never store business logic inside components.
- Frontend reacts only to events. Never infer backend state or calculate workflow logic. Render only what the backend publishes.

---

## Mock & Live Modes
- Every screen must support BOTH **Mock** and **Live** data pipelines without altering component rendering logic. Only the underlying data source changes.

---

## Theme Support
- Support Light and Dark modes. Never redesign either theme.

---

## Core Pages
- **Mission Control**: The primary demo page. Prioritize:
  `Planner ➔ Parallel Agents ➔ Conflict ➔ Tool Gate ➔ Decision`
- **Claims History**: Read-only history. No business logic.
- **System Health**: Displays backend health metrics without performing local calculations.
- **Settings**: Configuration settings only. No backend business logic.

---

## Code Standards
- Strict TypeScript.
- Functional Components with React Hooks only.
- No inline business logic.
- Meaningful component names and type-safe props.

---

## Performance
- Avoid unnecessary re-renders.
- Lazy load heavy components and memoize where appropriate.

---

## Backend Contract
- Always follow `shared/events.md` exactly.
- Do not invent event fields or modify event schemas.
- The backend is the single source of truth.
